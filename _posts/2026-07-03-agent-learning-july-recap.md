六月底到七月初，我集中试了一轮 Agent：写博客、改 Jeecg 前端、clone 项目读代码、配 Git 和 CMS……

没有变成「Agent 专家」，但有几条结论 ** likely 半年内还用得上**，写在这里当锚点。

## 1. Agent 的价值在「闭环」，不在「金句」

能 **读 → 改 → 跑 → 再看结果** 的任务，Agent 帮助最大。  
纯聊天能解决的，不必硬上 Agent。

## 2. Prompt 重要，但任务拆分更重要

把「做一个完整后台」拆成：

- 加 `admin/index.html`
- 写 `config.yml` 字段
- 更新 README 说明

每一步成功率都更高。**会拆任务，比会写华丽 prompt 实用。**

## 3. Git 是 Agent 时代的安全带

小步 commit、重要改动前 stash、private 仓库别乱给 Token——  
**版本控制是人和 Agent 协作的基础设施**，不是可选项。

## 4. 基础能力没被替代，反而更值钱

调试、读文档、问清楚需求、代码 review——  
Agent 越强，**「判断对不对」** 越关键。  
这和五月底那篇「AI 不会取代我」的判断是一致的，只是 Agent 把感受放大了。

## 5. 学习路线：少追名词，多练场景

我不打算先啃完所有 Agent 论文再动手。  
我的路线很简单：

```
选真实小任务（博客/CMS/小 bug）
    ↓
用 Agent 做一版
    ↓
自己 review + 记录翻车点
    ↓
写进博客（就是你现在看的这些）
```

## 接下来写什么

「Agent」这个分类我会继续更新，可能包括：

- MCP 第一次接入踩坑
- 多仓库工作区 + Agent 怎么不提交错项目
- 和实习项目结合的具体案例（能公开的部份）

如果你也在学 Agent，欢迎对照你的场景——**同一工具，不同人的「外包清单」可能完全不同**，这很正常。

七月继续练，慢慢写。
