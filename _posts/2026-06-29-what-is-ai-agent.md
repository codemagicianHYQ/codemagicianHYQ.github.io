之前我用 AI，基本就是：问一句、答一句。

直到开始用 Cursor、Claude Code 这类工具，我才发现有个新概念一直在耳边响：**Agent**。

我查了很多资料，说法不一。下面是我**现阶段**的理解，不装懂，只写我真正能用的那部分。

## Agent 和「聊天 AI」差在哪？

| 聊天 AI | Agent |
|---------|-------|
| 你问，它答 | 你给目标，它**拆步骤、调工具、执行** |
| 主要在对话框里 | 可能读文件、跑命令、改代码、提交 Git |
| 一次对话一轮 | 可以多轮自主推进，直到完成或卡住 |

打个比方：

- **聊天 AI** 像咨询顾问：告诉你怎么做  
- **Agent** 像实习生：你说「帮我把博客后台配好」，它会自己去改配置、加文件、甚至跑命令验证  

当然，**实习生也会犯错**，所以最后还是要你验收。

## 我接触 Agent 的几个场景

1. **写博客**：让它改 `_config.yml`、加 `_posts` 文章、配 CMS 后台  
2. **配环境**：SSH、Git 远程、TypeScript 报错排查  
3. **Clone 项目后跑起来**：看 `package.json`、解释为什么要先 `npm i`  

这些以前我要自己一步步查文档；现在有 Agent，它会**主动打开文件、搜索、执行**，我负责判断对不对。

## 我现在的朴素定义

> **Agent = 大模型 + 目标 + 工具权限 + 多步执行**

工具可能是：读文件、终端、浏览器、MCP 插件等。

没有工具，就还是聊天；有了工具并允许它用，才更像 Agent。

## 别把它神化

我踩过的坑：

- 让它改 Git 配置，结果改的是外层 `front` 仓库，不是博客仓库  
- 网络不稳时 push 失败，它也会误判  
- 有时候「说做好了」，其实还要我手动 push 或清缓存  

所以我的态度是：**Agent 是杠杆，不是替身。**
