很多人把 AI 当成「问答工具」，但在真实开发里，最有价值的是把它做成流程组件。

这篇文章分享一个简单思路：  
把 Git 提交流程拆成明确步骤，再让 AI 按规则执行，从而降低遗漏和返工。

## 可复用流程

1. 检查变更范围（`git status` + `git diff`）
2. 自动生成符合仓库风格的 commit message
3. 提交前跑最小测试，失败就停止
4. 输出可审阅摘要，便于回看和复盘

## 为什么有效

- 人脑更适合做判断，不适合反复做机械步骤
- 固化流程后，团队提交质量会明显更稳定
- 新成员也能更快进入统一协作节奏
