之前我用 AI，基本就是：问一句、答一句、复制粘贴。

直到开始用 Cursor 的 Agent、Claude Code 这类工具，我才发现：**Agent 和 Chat 不是同一类东西。**

## 我现在的粗浅理解

|      | Chat（对话）   | Agent（智能体）          |
| ---- | -------------- | ------------------------ |
| 模式 | 你问它答       | 你给目标，它自己拆步骤   |
| 工具 | mostly 文字    | 能读文件、跑命令、改代码 |
| 适合 | 查概念、写片段 | 改项目、排查、批量任务   |
| 风险 | 相对低         | 可能改错文件，要会审     |

Chat 像 **助教**；Agent 更像 **会动手的实习生**——能帮你打开仓库、搜代码、执行命令，但最后的责任还在你。

## 我踩过的认知坑

**坑 1：以为 Agent 什么都能一次搞定**

实际经常：第一步对了，后面跑偏，或者改了不该改的文件。

**坑 2：把 Agent 输出当最终结果**

现在我会强制自己：**Agent 改完 = 草稿，不是交付。** 必须自己 diff 一遍再 commit。

**坑 3：任务描述太模糊**

「帮我优化项目」几乎必翻车。  
改成「只改 `src/api/user.ts`，把重复请求合并，别动别的文件」成功率明显高。

## 我为什么开始专门学 Agent

实习里重复劳动多：改配置、写样板代码、查报错、整理 commit message。  
这些 Chat 也能帮一点，但 **Agent 能直接在仓库里操作**，省掉大量复制粘贴。

同时我也在练 Electron 小项目（比如桌面客户端），Agent 帮我读陌生代码结构快很多——**但业务逻辑还得自己判断。**

## 小结

Agent 不是魔法，是 **「大模型 + 工具调用 + 任务规划」** 的组合。
